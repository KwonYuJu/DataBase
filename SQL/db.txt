KEY란 : 테이블에서 고유하게 식별할 수 있는 필드
특징 2가지
1. 중복 X
2. NULL X

종류 3가지
1. 기본키(Primary Key)
: 테이블 내에서 각각 레코드를 식별하는 역할을 하는 필드
ex) 게시글의 ID
2. 후보키(Candidate Key)
: 기본키와의 차이는 여러개가 존재 할 수 있다.
ex) 주민번호, 휴대폰 번호, 이메일
3. 외래키(Foreign Key)
: 테이블이 두개 이상 일때
: 하나의 테이블에서 다른테이블의 기본키나 후보키를 참조하는 필드
- 외래키를 사용해서 두개 이상의 테이블을 연결할 수 있다.(관계형 DB)
- 1:1, 1:N, M:N 관계를 외래키를 통해서 표현한다.

DBMS(Management system)
: 데이터 베이스 관리 시스템 - MySQL, Oracle, sqlite3 등등...

서버가 DB에 요청하는 것을 Query라고 한다.
우리가 여기서 사용하는 기본 문법은 SQL
그리고 우리가 지금까지는 단일 테이블 구조만 배웠지만 앞으로는 관계형
테이블 구조까지 더 배울 예정
단일 테이블 구조, 관계형 테이블 구조 ---> 스키마 구조

그러면 모든 DB의 구조가 스키마 구조일까??
SQL(관계형 데이터 베이스)이 있으면 NoSQL(비 관계형 데이터 베이스)

1. SQL(관계형 데이터 베이스)
- 정형화된 스키마 구조가
- 데이터는 테이블, 행, 열 형태로 구조화
- ex) MySQL, Oracle 등등

2. NoSQL(비관계형 데이터 베이스)
- key-value 형(Redis)
- 문서형(MongoDB)
- 컬럼형(Cassandra)
- 그래프형(Neo4j)
: 대규모 분산 시스템, 빅테이터 처리아 자주 사용된다.

NoSQL 쓰는 이유??
: 성능적인 면에서 특정 유형의 작업에서 SQL보다 빠를 수 있다.
  (ex) 대용량의 데이터 처리)
